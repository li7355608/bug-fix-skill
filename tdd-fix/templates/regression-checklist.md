# 回归检查清单

修复完成后，逐项检查以下内容，确保修复没有引入新问题。

## 通用检查项

### 1. 配置项读取
- [ ] 修复是否涉及配置项（application.yml / application.properties）？
- [ ] 配置项的默认值是否合理？缺失时是否会导致异常？
- [ ] 多环境配置（dev / test / prod）是否兼容？

### 2. 数据库相关
- [ ] 是否改动了实体类（Entity）或数据传输对象（DTO）的字段？
- [ ] 数据库映射（MyBatis XML / JPA 注解）是否需要同步更新？
- [ ] SQL 语句是否受影响（字段名、条件、排序等）？
- [ ] 是否需要数据迁移或数据修复脚本？

### 3. 外部接口调用
- [ ] 修复是否涉及调用外部 API / RPC / 消息队列？
- [ ] 请求参数或响应处理逻辑是否变化？
- [ ] 超时、重试、熔断策略是否仍然有效？

### 4. 依赖注入
- [ ] 是否新增、删除或修改了 Spring Bean？
- [ ] 依赖注入方式（构造器 / @Autowired / @Resource）是否正确？
- [ ] 是否存在循环依赖风险？

### 5. 异常处理
- [ ] 异常处理链路是否完整（catch → log → 转换 → 返回）？
- [ ] 是否有异常被静默吞掉（空 catch 块）？
- [ ] 全局异常处理器（@ControllerAdvice）是否能正确捕获新增的异常类型？

### 6. 线程安全与并发
- [ ] 修改的代码是否在多线程环境下执行？
- [ ] 共享状态是否有适当的同步机制？
- [ ] 是否涉及缓存更新（本地缓存 / Redis）？缓存一致性是否受影响？

### 7. 接口契约
- [ ] Controller 层的请求参数或响应结构是否变化？
- [ ] 如果有变化，前端或调用方是否需要同步修改？
- [ ] API 文档（Swagger / OpenAPI）是否需要更新？

### 8. 日志与监控
- [ ] 关键路径是否有足够的日志输出？
- [ ] 修复后的日志级别是否合适（不要在正常路径打 ERROR）？

### 9. 编译与构建
- [ ] 项目能否正常编译通过？
- [ ] 是否有新增的编译警告？

### 10. 权限与安全
- [ ] 修复是否影响了接口权限校验逻辑（如 @PreAuthorize、拦截器、过滤器）？
- [ ] 数据查询的过滤条件是否受影响（如租户隔离、数据权限、行级过滤）？
- [ ] 是否涉及敏感数据的脱敏、加解密处理？
- [ ] 输入校验（参数合法性、SQL 注入、XSS 防护）是否仍然有效？

### 11. 已有测试
- [ ] 被修改类所在模块的已有测试是否全部通过？
- [ ] 如果改动涉及公共组件，依赖该组件的其他模块测试是否通过？

---

## 项目级补充检查项

> 如果项目上下文文件（`PROJECT-CONTEXT.md` / `CLAUDE.md` / `.cursorrules` / `.github/copilot-instructions.md`）中定义了业务级检查项（如特定模块的联动检查、特殊配置验证等），**必须追加到此清单末尾并逐项执行**。
